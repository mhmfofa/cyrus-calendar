ng-cyrus-calendar

Installation

npm install ng-cyrus-calendar

The package bundles the IRANSans font internally for Persian/Arabic calendars — no CDN or extra font setup needed.

If you use the Shamsi (Jalali) calendar, jalali-moment is also required:

npm install jalali-moment

Quick Start

Add the directive cyrus-calendar to any <input> element and place a <calendar-popup> next to it. Pass the input's template reference via [input] and the directive reference via [directive].

<!-- Template -->
<input
  type="text"
  #myInput
  #cal="cyrusCalendar"
  cyrus-calendar
  [calendar-type]="DatePickerType.Shamsi"
  [(ngModel)]="selectedDate"
/>

<calendar-popup
  [input]="myInput"
  [directive]="cal"
  [(ngModel)]="selectedDate">
</calendar-popup>

Optional: global configuration

Use CyrusCalendarConfigService to set defaults once at the root level:

// app.module.ts or app.config.ts
import { CyrusCalendarConfigService } from 'ng-cyrus-calendar';

// In providers or in the constructor:
configService.defaultCalendarType.set(DatePickerType.Gregorian);
configService.defaultFormat.set('MM/dd/yyyy');

Directive: cyrus-calendar

Applied directly on <input> elements. Export as #myRef="cyrusCalendar" to pass to <calendar-popup [directive]="myRef">.

Input Type Default Description
[calendar-type] DatePickerType 'imperial' Sets the active calendar system. Automatically updates the placeholder language.
[disable-weekends] boolean false Prevents selection of weekend days (Fri/Sat for Persian, Sat/Sun for Gregorian).
[disable-past-days] boolean false Prevents selection of any date before today.
[placeholder] string auto Custom placeholder text. When omitted, auto-sets to "انتخاب تاریخ" for Persian or "Select a date" for Gregorian.

Component: <calendar-popup>

Implements ControlValueAccessor — works with both ngModel and reactive formControl / formControlName.

Input Type Default Description
[input] HTMLInputElement — Required. Template reference of the attached input element.
[directive] CyrusCalendarDirective null Recommended. Pass the directive reference to automatically inherit calendar-type, disable-weekends, and disable-past-days. Avoids duplicate bindings.
[calendar-type] DatePickerType 'shamsi' Calendar system. Ignored when [directive] is provided.
[format] string 'yyyy/MM/dd' Display format shown inside the input element. Uses slashes — e.g. 1403/06/15. Does not affect the stored/emitted value.
[value-format] string 'yyyy-MM-dd' Structure of the emitted Gregorian date string — e.g. 2026-03-08. Only affects separators / field order. The emitted date is always Gregorian regardless of the displayed calendar type.
[min] string null Minimum selectable date. Must match the [format] pattern.
[max] string null Maximum selectable date. Must match the [format] pattern.
[disable-weekends] boolean false Disable weekends. Overridden by [directive] value when provided.
[disable-past-days] boolean false Disable past dates (sets min to today). Overridden by [directive].
[from-tomorow] boolean false Only allow selection from tomorrow onwards (sets min to tomorrow).
[date] boolean true Show the date picker panel.
[time] boolean false Show the time picker panel (hour/minute/second).
[time-format] string 'hh:mm:ss' Format for the time part. Only used when [time]="true". The full stored value becomes [format]T[time-format] and the input displays [format] - [time-format]. Tokens: hh (hour), mm (minute), ss (second).
[multiple] boolean false Allow selecting multiple dates (comma-separated value string).
[range] boolean false Allow selecting a date range (start and end date).
[options] DatePickerOptions — Advanced configuration object. Overrides individual inputs when provided.
📌 Value is always Gregorian. The value emitted to ngModel / formControl is always in the Gregorian calendar, regardless of which calendar type is displayed to the user. For example, selecting 1404/12/17 in Shamsi or 2584/12/17 in Imperial both emit 2026-03-08. This means you can send the value directly to REST APIs or databases without any extra conversion on your side.
Methods
MethodDescription
open()Programmatically open the calendar popup.
close()Programmatically close the calendar popup.
writeValue(value)Set the displayed date value (standard CVA method).

Enum: DatePickerType

ValueStringDescription
DatePickerType.Shamsi'shamsi'Jalali / Solar Hijri calendar (1400s). RTL layout, Persian digits and month names.
DatePickerType.Imperial'imperial'Imperial / Shahanshahi calendar (2500s). RTL layout, Persian digits and month names.
DatePickerType.Gregorian'gregorian'Standard Gregorian calendar. LTR layout, Monday as first day, English month names.

Date Format Tokens

The [format] input uses the following tokens (case-sensitive):

TokenOutputExample
yyyy4-digit year1403, 2025
yy2-digit year03, 25
MM2-digit month (zero-padded)01 – 12
MMonth without leading zero1 – 12
dd2-digit day (zero-padded)01 – 31
dDay without leading zero1 – 31
hh24-hour (zero-padded)00 – 23
mmMinutes (zero-padded)00 – 59
ssSeconds (zero-padded)00 – 59

Default format: yyyy/MM/dd

Live Examples

1 — Basic Shamsi (Jalali) Picker

The default calendar type. No extra configuration needed.

Value: {{ example1Value() || '—' }} 
<input
  type="text"
  #myInput
  #cal="cyrusCalendar"
  class="form-control"
  cyrus-calendar
  [calendar-type]="DatePickerType.Shamsi"
  [(ngModel)]="selectedDate"
  autocomplete="off"
  inputmode="none"
/>

<calendar-popup
  [input]="myInput"
  [directive]="cal"
  [(ngModel)]="selectedDate">
</calendar-popup>

4 — Date & Time Picker

Add [time]="true" to the popup to enable time selection.

Value: {{ example4Value() || '—' }} 
<calendar-popup
  [input]="myInput"
  [directive]="cal"
  [time]="true"
  [(ngModel)]="selectedDateTime">
</calendar-popup>

<!-- Optional: customise the time part of the format (default: hh:mm:ss) -->
<calendar-popup [time]="true" [time-format]="'hh:mm'" ...></calendar-popup>

<!-- Value stored & emitted: 1403/06/15T14:30:00  -->
<!-- Input shows:              1403/06/15 - 14:30:00 -->

5 — Calendar Switcher (value preserved on type change)

When the user switches calendar type, the existing value is automatically converted to the new system — no data is lost.

Value: {{ example5Value() || '—' }}

calendarType = signal<DatePickerType>(DatePickerType.Imperial);

// Template
<input
  cyrus-calendar
  [calendar-type]="calendarType()"
  ...
/>
